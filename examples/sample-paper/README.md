# 示例项目：学术论文

这是一个使用Specify Writer七阶段工作流程撰写学术论文的完整示例。

## 项目信息

- **论文题目**: 大语言模型在数据治理中的应用研究
- **类型**: 硕士学位论文（MEM专业）
- **目标字数**: 30,000字
- **章节数**: 6章
- **研究周期**: 2025-09-01 至 2025-11-30

## 使用的命令序列

### 1. 初始化项目

```bash
/writer:init "LLM数据治理研究" "examples/sample-paper" "学术论文"
```

**输出**: 创建符合学术规范的项目结构

### 2. 文献调研

```bash
/writer:1-research
```

**调研内容**:
- 数据治理理论框架（DAMA-DMBOK 2.0）
- 大语言模型技术综述
- LLM在企业应用的案例研究
- 相关学术论文（近5年，50+篇）
- 行业标准和最佳实践

**调研方法**:
- 文献检索（知网、Web of Science、Google Scholar）
- 行业报告分析
- 专家访谈
- 企业案例调研

**输出文件**: `research-report.md`

### 3. 论文设计

```bash
/writer:2-design
```

**设计要点**:
- **研究方法**: 定性研究 + 案例分析
- **论文结构**: IMRaD格式（引言-方法-结果-讨论）
- **写作风格**: 学术正式、客观严谨
- **引用格式**: GB/T 7714-2015
- **图表规范**: 三线表、矢量图

**输出文件**: `design-document.md`

### 4. 创作宪法

```bash
/writer:3-constitution
```

**学术规范**:
1. 严格遵守学术诚信，杜绝抄袭
2. 所有引用必须标注来源
3. 数据和结论必须有充分支撑
4. 保持客观中立的学术态度
5. 使用规范的学术语言
6. 图表必须清晰、准确、规范
7. 符合MEM专业学位论文标准

**输出文件**: `constitution.md`

### 5. 论文大纲

```bash
/writer:4-outline -Chapters 6
```

**大纲结构**:

**第一章：绪论**
- 1.1 研究背景与意义
- 1.2 国内外研究现状
- 1.3 研究内容与方法
- 1.4 论文组织结构

**第二章：理论基础**
- 2.1 数据治理理论
- 2.2 大语言模型技术
- 2.3 相关概念界定

**第三章：LLM在数据治理中的应用框架**
- 3.1 应用场景分析
- 3.2 技术架构设计
- 3.3 实施路径规划

**第四章：案例研究**
- 4.1 案例选择与方法
- 4.2 案例实施过程
- 4.3 案例效果评估

**第五章：问题与对策**
- 5.1 实施中的主要问题
- 5.2 解决方案与建议
- 5.3 未来发展趋势

**第六章：结论与展望**
- 6.1 研究结论
- 6.2 创新点
- 6.3 研究局限
- 6.4 未来研究方向

**输出文件**: `outline.md`

### 6. 合规性审查

```bash
/writer:5-validate
```

**验证项目**:
- ✅ 符合MEM专业学位论文标准
- ✅ 引用格式规范
- ✅ 逻辑结构完整
- ✅ 研究方法科学
- ✅ 数据来源可靠
- ✅ 结论有充分支撑

**输出文件**: `validation-report.md`

### 7. 章节撰写

```bash
# 撰写各章节
/writer:6-draft -Chapter 1  # 绪论
/writer:6-draft -Chapter 2  # 理论基础
/writer:6-draft -Chapter 3  # 应用框架
/writer:6-draft -Chapter 4  # 案例研究
/writer:6-draft -Chapter 5  # 问题与对策
/writer:6-draft -Chapter 6  # 结论与展望
```

**输出文件**: `chapter-1.md` 至 `chapter-6.md`

### 8. 最终审查与格式化

```bash
/writer:7-finalize
```

**最终输出**:
- `final-document.md` - 完整论文（Markdown）
- `final-document.pdf` - PDF版本（符合提交要求）
- `final-document.docx` - Word版本（用于修改）

## 示例内容预览

### 摘要

```markdown
# 摘要

随着数字化转型的深入推进，企业面临着日益复杂的数据治理挑战。传统的数据治理
方法在处理海量、多源、异构数据时显得力不从心。大语言模型（LLM）作为人工智能
领域的重要突破，为数据治理提供了新的技术路径。

本研究基于DAMA-DMBOK 2.0框架，系统分析了大语言模型在数据治理中的应用场景，
包括元数据管理、数据质量评估、数据分类分级等关键领域。通过对某大型企业的
案例研究，验证了LLM技术在提升数据治理效率和质量方面的显著效果。

研究结果表明：（1）LLM能够自动化处理大量重复性数据治理任务，效率提升60%以上；
（2）基于LLM的智能推荐系统可以显著提高数据治理决策的准确性；（3）LLM在数据
语义理解方面具有独特优势，有助于打破数据孤岛。

本研究的创新点在于：首次提出了"LLM增强的端到端数据治理框架"，并通过实际
案例验证了其可行性和有效性。研究成果对企业数字化转型和数据治理实践具有
重要的参考价值。

**关键词**: 大语言模型；数据治理；DAMA-DMBOK；人工智能；企业数字化转型
```
