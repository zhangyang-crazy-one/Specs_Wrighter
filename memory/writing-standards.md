## 1. 通用规范

### 1.1 文件编码
- **强制要求**: 所有文件使用 UTF-8 with BOM 编码
- **中文支持**: 完整支持中英文双语内容
- **换行符**: 统一使用 LF (Unix风格)

### 1.2 文件命名
- **项目文件**: 使用小写字母和连字符,如 `my-project.md`
- **配置文件**: 使用小写字母和下划线,如 `project_config.json`
- **风格文档**: 使用项目名前缀,如 `[project-name]_style.md`

### 1.3 目录结构
```
project-name/
├── main.md                    # 主文档
├── outline.md                 # 大纲
├── project-config.json        # 配置文件
├── chapters/                  # 章节目录(可选)
│   ├── chapter01.md
│   └── chapter02.md
└── assets/                    # 资源目录(可选)
    ├── images/
    └── references/
```

## 2. Markdown规范

### 2.1 标题层级
- **一级标题**: 仅用于文档标题
- **二级标题**: 用于主要章节
- **三级标题**: 用于子章节
- **四级及以下**: 用于细分内容

示例:
```markdown
# 文档标题

## 第一章

### 1.1 第一节

#### 1.1.1 细分内容
```

### 2.2 段落格式
- 段落之间空一行
- 每行不超过100个字符(建议)
- 中英文之间添加空格

### 2.3 列表格式
- **无序列表**: 使用 `-` 符号
- **有序列表**: 使用数字加点 `1.`
- **嵌套列表**: 使用2个空格缩进

示例:
```markdown
- 第一项
  - 子项1
  - 子项2
- 第二项

1. 第一步
2. 第二步
```

### 2.4 代码块
- 使用三个反引号包裹
- 指定语言类型以启用语法高亮

示例:
````markdown
```python
def hello():
    print("Hello, World!")
```
````

### 2.5 引用格式
- 使用 `>` 符号
- 多行引用每行都加 `>`

示例:
```markdown
> 这是一段引用文本。
> 可以跨越多行。
```

## 3. 标点符号规范

### 3.1 中文标点
- 使用全角标点: 。,、;:?!""''()《》【】
- 省略号使用: ……
- 破折号使用: ——

### 3.2 英文标点
- 使用半角标点: .,;:?!"'()-
- 省略号使用: ...
- 破折号使用: --

### 3.3 中英文混排
- 中英文之间添加空格
- 中文与数字之间添加空格
- 英文标点后添加空格

示例:
```
正确: 我使用 Python 3.9 版本进行开发。
错误: 我使用Python3.9版本进行开发。
```

## 4. 引用和参考文献

### 4.1 行内引用
- 学术论文: 使用 `[作者, 年份]` 格式
- 技术文章: 使用脚注或超链接

示例:
```markdown
根据研究 [Smith, 2020],数据治理对企业至关重要。

详见[官方文档](https://example.com)。
```

### 4.2 参考文献列表
- 放在文档末尾
- 使用二级标题 `## 参考文献`
- 按字母顺序或引用顺序排列

示例:
```markdown
## 参考文献

[1] Smith, J. (2020). Data Governance Best Practices. Publisher.
[2] 张三. (2021). 数据管理实践. 出版社.
```

## 5. 图表规范

### 5.1 图片
- 使用相对路径引用
- 添加替代文本(alt text)
- 图片文件名使用小写和连字符

示例:
```markdown
![数据治理框架](assets/images/data-governance-framework.png)
```

### 5.2 表格
- 使用Markdown表格语法
- 表头与内容之间使用分隔线
- 对齐方式: 左对齐(默认)、居中、右对齐

示例:
```markdown
| 列1 | 列2 | 列3 |
|-----|:---:|----:|
| 左对齐 | 居中 | 右对齐 |
```

### 5.3 图表编号
- 图: `图 X-Y: 标题`
- 表: `表 X-Y: 标题`
- X为章节号,Y为序号

## 6. 版本控制

### 6.1 版本号规范
- 使用语义化版本: `主版本.次版本.修订版本`
- 主版本: 重大变更
- 次版本: 功能增加
- 修订版本: 错误修复

### 6.2 变更日志
- 在文档末尾添加 `## 更新日志`
- 记录每个版本的主要变更

示例:
```markdown
## 更新日志

### v1.1.0 (2025-01-15)
- [新增] 添加第三章内容
- [修改] 优化第一章结构

### v1.0.0 (2025-01-02)
- [新增] 初始版本发布
```

## 9. 禁止事项

### 9.1 符号使用
- ❌ 禁止使用emoji表情符号
- ❌ 禁止使用特殊Unicode字符(除中文外)
- ❌ 禁止使用制表符(Tab),使用空格代替

### 9.2 内容规范
- ❌ 禁止抄袭和剽窃
- ❌ 禁止使用攻击性语言
- ❌ 禁止泄露敏感信息

### 9.3 格式规范
- ❌ 禁止混用不同的标题符号
- ❌ 禁止混用不同的列表符号
- ❌ 禁止使用HTML标签(除非必要)
