## 1. 学术写作风格 (Academic Style)

### 1.1 研究论文风格

**语言特征**:
- **语调**: 正式、客观、严谨
- **人称**: 第三人称为主,避免第一人称
- **时态**: 现在时(陈述事实)、过去时(描述实验)
- **语气**: 陈述性、论证性

**词汇特征**:
- 使用专业术语和学术词汇
- 避免口语化表达
- 使用精确的量化描述

**句式特征**:
- 句子结构复杂,逻辑严密
- 使用连接词表达因果、转折关系
- 平均句长: 20-30字

**示例段落**:
```
数据治理是组织对数据资产进行管理和控制的系统性方法。根据DAMA-DMBOK2框架,
数据治理包括数据质量管理、元数据管理、数据安全等多个知识领域。本研究通过
案例分析方法,探讨了企业在实施数据治理过程中面临的主要挑战及应对策略。
```

---

### 1.2 文献综述风格

**语言特征**:
- **语调**: 客观、综合性
- **人称**: 第三人称
- **时态**: 现在完成时(综述已有研究)
- **语气**: 评述性、批判性

**词汇特征**:
- 使用"研究表明"、"学者认为"等评述性词汇
- 引用多位学者观点
- 使用比较和对比词汇

**句式特征**:
- 使用并列和递进结构
- 频繁使用引用和转述
- 平均句长: 25-35字

**示例段落**:
```
关于数据治理成熟度评估,学术界已有丰富的研究成果。Smith等人(2020)提出了
基于能力成熟度模型的评估框架,而Zhang(2021)则从组织文化角度探讨了数据
治理的成功因素。这些研究为本文提供了重要的理论基础。
```

## 2. 文章写作风格 (Article Style)

### 2.1 技术文章风格

**语言特征**:
- **语调**: 专业但易懂
- **人称**: 第一人称复数(我们)或第二人称(你)
- **时态**: 现在时为主
- **语气**: 说明性、指导性

**词汇特征**:
- 使用技术术语但提供解释
- 适当使用类比和比喻
- 平衡专业性和可读性

**句式特征**:
- 句子简洁明了
- 使用短句和中等长度句子
- 平均句长: 15-20字

**示例段落**:
```
在Python中,我们可以使用pandas库来处理数据。首先,导入必要的库:
import pandas as pd。然后,读取CSV文件:df = pd.read_csv('data.csv')。
这样就完成了数据的加载,接下来可以进行数据分析了。
```

---

### 2.2 博客文章风格

**语言特征**:
- **语调**: 轻松、亲切、对话式
- **人称**: 第一人称(我)或第二人称(你)
- **时态**: 现在时为主
- **语气**: 分享性、互动性

**词汇特征**:
- 使用日常词汇
- 适当使用网络流行语(但不过度)
- 避免过于专业的术语

**句式特征**:
- 句子短小精悍
- 使用问句增加互动感
- 平均句长: 10-15字

**示例段落**:
```
你有没有遇到过这种情况?写代码写到一半,突然忘记了某个函数的用法。
别担心,这很正常!今天我就来分享几个提高编程效率的小技巧。相信我,
学会这些,你的开发速度至少能提升30%!
```

## 3. 创意写作风格 (Creative Style)

### 3.1 小说风格

**语言特征**:
- **语调**: 生动、富有感染力
- **人称**: 第一人称或第三人称
- **时态**: 过去时为主(叙事)
- **语气**: 叙述性、描写性

**词汇特征**:
- 使用形象化词汇
- 运用修辞手法(比喻、拟人等)
- 注重感官描写

**句式特征**:
- 句式多样化
- 长短句结合
- 平均句长: 15-25字

**示例段落**:
```
夜幕降临,城市的灯光一盏盏亮起,像是天空中坠落的星辰。李明站在窗前,
望着楼下熙熙攘攘的人群,心中涌起一股莫名的孤独感。他知道,今晚又将是
一个不眠之夜。
```

---

### 3.2 剧本风格

**语言特征**:
- **语调**: 简洁、动作化
- **人称**: 第三人称(舞台指示)
- **时态**: 现在时
- **语气**: 指示性、对话性

**词汇特征**:
- 使用动作动词
- 对话口语化
- 舞台指示简洁明确

**句式特征**:
- 对话短促有力
- 舞台指示使用括号
- 平均句长: 8-12字

**示例段落**:
```
INT. 咖啡厅 - 下午

李明坐在靠窗的位置,手里拿着一杯咖啡。

李明
(看着窗外)
她不会来了。

王芳从门口走进来,四处张望。

王芳
(看到李明,走过去)
抱歉,我来晚了。
```

---

### 3.3 诗歌风格

**语言特征**:
- **语调**: 抒情、意象化
- **人称**: 第一人称为主
- **时态**: 灵活运用
- **语气**: 情感性、象征性

**词汇特征**:
- 使用意象词汇
- 注重音韵和节奏
- 运用象征和隐喻

**句式特征**:
- 分行排列
- 注重韵律和节奏
- 句长灵活

**示例段落**:
```
月光如水
洒在静谧的湖面
涟漪轻轻荡开
带走了
我心中的思念
```

## 4. 专业文档风格 (Professional Style)

### 4.1 技术规格说明书风格

**语言特征**:
- **语调**: 精确、规范
- **人称**: 第三人称或被动语态
- **时态**: 现在时
- **语气**: 说明性、规定性

**词汇特征**:
- 使用标准化术语
- 避免歧义表达
- 使用量化描述

**句式特征**:
- 句子结构规范
- 使用列表和表格
- 平均句长: 15-20字

**示例段落**:
```
系统要求:
- 操作系统: Windows 10或更高版本
- 内存: 最低8GB RAM
- 硬盘空间: 至少50GB可用空间
- 网络: 宽带互联网连接

性能指标:
- 响应时间: 不超过2秒
- 并发用户数: 支持1000+
- 可用性: 99.9%
```

---

### 4.2 商业提案风格

**语言特征**:
- **语调**: 专业、说服性
- **人称**: 第一人称复数(我们)
- **时态**: 现在时和将来时
- **语气**: 建议性、展望性

**词汇特征**:
- 使用商业术语
- 强调价值和收益
- 使用积极正面词汇

**句式特征**:
- 句子清晰有力
- 使用数据支撑论点
- 平均句长: 15-20字

**示例段落**:
```
我们提出的数据治理解决方案将为贵公司带来以下价值:
1. 提升数据质量,减少错误率30%以上
2. 降低数据管理成本,预计节省20%的运营费用
3. 增强数据安全性,确保合规性

预计投资回报周期为18个月,长期收益显著。
```

## 5. 风格混合指南

### 5.1 何时混合风格

某些文档可能需要混合多种风格:
- **技术博客**: 技术文章 + 博客风格
- **科普文章**: 学术风格 + 文章风格
- **创意非虚构**: 创意风格 + 文章风格

### 5.2 混合原则

1. **明确主导风格**: 确定文档的主要风格
2. **平滑过渡**: 在风格转换处使用过渡段落
3. **保持一致**: 同一部分内保持风格统一
4. **服务目的**: 风格选择服务于内容目标

## 6. 风格自定义

### 6.1 创建自定义风格

使用 `writer.style` 命令分析参考文本,创建自定义风格文档:

```yaml
project: my-project
language: 中文
tone: 5  # 1-5,5最正式
person: 第三人称
tense: 现在时
voice: 主动语态
avg_sentence_length: 20
vocabulary_complexity: 中等
```

### 6.2 风格演化

随着项目进展,风格可能需要调整:
- 定期审查风格一致性
- 根据反馈调整风格
- 记录风格变更历史
