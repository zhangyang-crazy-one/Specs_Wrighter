# 创作宪法文件编写原则

本规则文件定义了如何创建和验证"创作宪法文件"（Constitution File），这是七阶段写作工作流程中的核心文档。

---

## 什么是创作宪法文件

### 定义

创作宪法文件是一个定义整个创作过程中**不可违背的核心原则和约束条件**的文档，类似于国家宪法对法律的约束作用。

### 作用

1. **统一性保障**: 确保整个作品的主题、风格、价值观保持一致
2. **质量控制**: 提供明确的评判标准，用于合规性审查
3. **创作指南**: 为内容编写提供清晰的方向和边界
4. **冲突解决**: 当出现创作分歧时，以宪法文件为最高准则

---

## 宪法文件必须包含的章节

### 1. 核心主题和价值观

**定义**: 作品要传达的核心思想和价值取向

**示例（科幻小说）**:
```markdown
## 核心主题和价值观

### 核心主题
- 人类与人工智能的共存
- 技术进步与人性保留的平衡
- 个体自由与集体利益的冲突

### 价值观
- 尊重生命的多样性（包括人工智能生命）
- 批判技术乐观主义
- 强调人性中的同理心和道德选择
```

**示例（学术论文）**:
```markdown
## 核心主题和价值观

### 核心主题
- 数据隐私保护的法律框架
- 技术发展与隐私权的平衡

### 价值观
- 客观中立的学术立场
- 基于实证研究的论证
- 尊重不同学术观点
```

### 2. 不可违背的创作原则

**定义**: 在任何情况下都必须遵守的创作规则

**示例（小说）**:
```markdown
## 不可违背的创作原则

1. **叙事视角一致性**: 全文使用第三人称限知视角，不得切换
2. **时间线连贯性**: 严格遵循线性时间线，禁止闪回或闪前
3. **科学准确性**: 所有科技设定必须基于现有科学理论的合理延伸
4. **角色行为逻辑**: 角色行为必须符合其性格设定和动机
5. **语言风格**: 保持冷静、克制的叙事语言，避免过度煽情
```

**示例（学术论文）**:
```markdown
## 不可违背的创作原则

1. **学术规范**: 严格遵循APA第7版引用格式
2. **论证逻辑**: 所有论点必须有充分的文献支持或实证数据
3. **客观性**: 避免主观臆断和情绪化表达
4. **原创性**: 所有内容必须原创，引用必须明确标注
5. **结构完整**: 必须包含摘要、引言、文献综述、方法、结果、讨论、结论
```

### 3. 角色设定规则（如适用）

**定义**: 对角色创建和发展的约束条件

**示例（小说）**:
```markdown
## 角色设定规则

### 主要角色
- **数量限制**: 主要角色不超过5人
- **性格复杂性**: 每个主要角色必须有至少一个性格缺陷
- **成长弧线**: 每个主要角色必须经历明显的性格转变

### 角色关系
- **冲突必要性**: 主要角色之间必须存在价值观冲突
- **关系演变**: 角色关系必须随情节发展而变化
- **禁止完美角色**: 不得出现全知全能、毫无缺点的角色

### 角色行为
- **动机合理性**: 所有重大行为必须有明确的动机
- **一致性**: 角色行为必须符合其性格设定
- **成长逻辑**: 性格转变必须有充分的情节铺垫
```

**示例（传记）**:
```markdown
## 角色设定规则

### 主人公刻画
- **真实性**: 所有事件必须基于历史事实
- **客观性**: 避免过度美化或丑化
- **多面性**: 展现人物的复杂性和矛盾性

### 配角处理
- **历史准确**: 配角的言行必须符合历史记录
- **功能性**: 配角必须服务于主人公的故事线
```

### 4. 情节发展约束条件

**定义**: 对情节设计和推进的限制

**示例（小说）**:
```markdown
## 情节发展约束条件

### 情节结构
- **三幕结构**: 严格遵循三幕剧结构（建立-对抗-解决）
- **节奏控制**: 每5章必须有一个重大情节转折
- **高潮位置**: 高潮必须出现在全书80%-90%位置

### 情节逻辑
- **因果关系**: 所有重大事件必须有明确的因果链
- **伏笔回收**: 所有伏笔必须在后续章节中回收
- **禁止突兀**: 禁止使用"天降神兵"式的情节解决方案

### 冲突设计
- **冲突升级**: 冲突必须逐步升级，不得突然降级
- **多层冲突**: 必须包含外部冲突和内部冲突
- **冲突解决**: 冲突解决必须符合角色性格和故事逻辑
```

**示例（剧本）**:
```markdown
## 情节发展约束条件

### 场景设计
- **场景数量**: 全剧不超过15个场景
- **场景转换**: 场景转换必须有明确的时间或空间标记
- **舞台可行性**: 所有场景必须在舞台上可实现

### 对话规则
- **自然性**: 对话必须符合角色身份和时代背景
- **推进性**: 对话必须推进情节或揭示角色性格
- **简洁性**: 避免冗长的独白（除非有特殊戏剧效果）
```

### 5. 风格一致性要求

**定义**: 对语言风格、叙事手法的统一要求

**示例（小说）**:
```markdown
## 风格一致性要求

### 语言风格
- **叙事语言**: 冷静、克制、精确
- **对话风格**: 简洁、自然、符合角色身份
- **描写手法**: 以视觉描写为主，避免过度的心理描写

### 修辞手法
- **允许使用**: 隐喻、象征、对比
- **谨慎使用**: 夸张、拟人
- **禁止使用**: 过度华丽的辞藻、陈词滥调

### 叙事节奏
- **章节长度**: 每章3000-5000字
- **场景切换**: 每章不超过3个场景
- **时间跨度**: 每章时间跨度不超过24小时（除特殊情况）
```

**示例（学术论文）**:
```markdown
## 风格一致性要求

### 语言风格
- **学术性**: 使用规范的学术语言
- **客观性**: 避免第一人称和主观判断
- **精确性**: 使用准确的专业术语

### 论证风格
- **逻辑性**: 论证必须严密、有层次
- **实证性**: 论点必须有数据或文献支持
- **批判性**: 对现有研究进行批判性分析

### 格式规范
- **引用格式**: 严格遵循APA第7版
- **图表格式**: 所有图表必须有编号和标题
- **章节编号**: 使用统一的编号系统
```

### 6. 禁止事项清单

**定义**: 明确列出在创作过程中绝对不能出现的内容或做法

**示例（小说）**:
```markdown
## 禁止事项清单

### 内容禁止
- ❌ 违反科学常识的设定（除非有合理解释）
- ❌ 角色突然性格大变（没有铺垫）
- ❌ 使用"原来这一切都是梦"式结局
- ❌ 出现与主题无关的冗长描写
- ❌ 使用陈词滥调和俗套情节

### 风格禁止
- ❌ 切换叙事视角
- ❌ 使用过度煽情的语言
- ❌ 插入作者评论或说教
- ❌ 使用网络流行语或时代性过强的词汇

### 结构禁止
- ❌ 跳过重要情节转折
- ❌ 留下未解决的重要伏笔
- ❌ 突然引入新的主要角色（后半部分）
```

**示例（学术论文）**:
```markdown
## 禁止事项清单

### 内容禁止
- ❌ 未经验证的数据或结论
- ❌ 抄袭或不当引用
- ❌ 主观臆断和情绪化表达
- ❌ 与主题无关的内容

### 格式禁止
- ❌ 不规范的引用格式
- ❌ 无标题或编号的图表
- ❌ 不一致的术语使用
- ❌ 口语化表达
```

## 创建宪法文件的流程

### 步骤1: 分析调研结果

- 阅读 `research-report.md`
- 提取核心主题和关键信息
- 确定写作类型和目标受众

### 步骤2: 分析设计文档

- 阅读 `style-guide.md`
- 阅读 `background.md`（如适用）
- 阅读 `worldview.md`（如适用）
- 提取风格要求和背景设定

### 步骤3: 定义核心原则

- 基于调研和设计，定义核心主题和价值观
- 列出不可违背的创作原则
- 确保原则具体、可执行、可验证

### 步骤4: 设定约束条件

- 根据写作类型，设定角色、情节、风格的约束
- 确保约束条件合理、不过度限制创作
- 平衡创作自由和质量控制

### 步骤5: 列出禁止事项

- 明确列出绝对不能出现的内容
- 基于常见错误和陷阱
- 确保清晰、无歧义

### 步骤6: 审查和完善

- 检查宪法文件的完整性
- 确保所有章节都已包含
- 验证原则和约束的可执行性

## 验证宪法文件的标准

### 完整性检查

- [x] 包含所有必需章节
- [x] 每个章节都有具体内容
- [x] 没有空白或占位符

### 明确性检查

- [x] 所有原则都清晰、具体
- [x] 所有约束都可执行、可验证
- [x] 没有模糊或歧义的表述

### 一致性检查

- [x] 宪法文件与调研结果一致
- [x] 宪法文件与设计文档一致
- [x] 各章节之间没有矛盾

### 可行性检查

- [x] 约束条件不过度限制创作
- [x] 原则可以在实际创作中遵守
- [x] 禁止事项合理、必要

## 使用宪法文件的最佳实践

### 1. 在大纲编写时

- 每个章节设计都参考宪法文件
- 确保情节发展符合约束条件
- 检查是否违反禁止事项

### 2. 在合规性审查时

- 使用宪法文件作为评判标准
- 逐项检查大纲是否符合原则
- 记录所有不符合项

### 3. 在内容编写时

- 开始编写前重读宪法文件
- 编写过程中定期对照检查
- 完成后进行自查

### 4. 在最终审查时

- 全文检查是否符合宪法文件
- 验证主题和价值观的一致性
- 确认没有违反禁止事项

## 常见错误和避免方法

### 错误1: 宪法文件过于宽泛

**症状**: 原则和约束过于笼统，无法指导实际创作

**避免方法**:
- 使用具体、可量化的标准
- 提供明确的示例
- 避免使用"尽量"、"可能"等模糊词汇

### 错误2: 宪法文件过于严格

**症状**: 约束条件过多，限制创作自由

**避免方法**:
- 只设定核心的、必要的约束
- 允许一定的创作灵活性
- 区分"必须"和"建议"

### 错误3: 宪法文件与设计文档矛盾

**症状**: 宪法文件的要求与风格设计或背景设定冲突

**避免方法**:
- 创建前仔细阅读所有前置文档
- 确保宪法文件是前置文档的总结和提炼
- 发现矛盾时及时修改

### 错误4: 宪法文件缺少关键章节

**症状**: 遗漏重要的约束条件或原则

**避免方法**:
- 使用模板确保完整性
- 根据写作类型调整章节
- 创建后进行完整性检查

## 不同写作类型的宪法文件重点

### 学术论文
- 重点: 学术规范、论证逻辑、引用格式
- 可省略: 角色设定规则

### 小说
- 重点: 角色设定、情节发展、风格一致性
- 必须包含: 所有章节

### 剧本
- 重点: 场景设计、对话规则、舞台可行性
- 特殊要求: 时间和空间约束

### 传记
- 重点: 真实性、客观性、历史准确性
- 特殊要求: 事实核查标准

### 诗歌
- 重点: 韵律规则、意象使用、主题表达
- 可省略: 情节发展约束
