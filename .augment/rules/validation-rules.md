# 合规性审查规则

本规则文件定义了如何对大纲和内容进行合规性审查，确保符合创作宪法文件的要求。

---

## 合规性审查的目的

### 核心目标

1. **确保一致性**: 验证大纲和内容是否符合宪法文件定义的核心原则
2. **发现问题**: 及早发现偏离主题、风格不一致等问题
3. **质量保障**: 在内容编写前确保大纲的质量
4. **节省时间**: 避免编写大量不符合要求的内容后再返工

### 审查时机

- **阶段五**: 大纲编写完成后，内容编写前（强制）
- **阶段六**: 每章编写完成后（可选，建议）
- **阶段七**: 最终审查时（强制）

---

## 四大检查维度

### 1. 主题一致性检查

**定义**: 验证大纲和内容是否始终围绕宪法文件定义的核心主题展开

**检查项**:

#### 1.1 核心主题贯穿性
- [ ] 每个章节都与核心主题相关
- [ ] 没有偏离主题的内容
- [ ] 主题在全文中保持一致

**检查方法**:
```
1. 提取宪法文件中的核心主题
2. 逐章检查是否与核心主题相关
3. 标记偏离主题的章节
4. 评估偏离程度（轻微/严重）
```

**示例（小说）**:
```markdown
核心主题: 人类与AI的共存

✅ 通过:
- 第3章: 主角与AI助手建立信任关系
- 第7章: 人类社会对AI的恐惧和排斥
- 第12章: AI获得自我意识后的道德困境

❌ 不通过:
- 第5章: 主角的童年回忆（与主题无关）
- 第9章: 详细描写未来城市的建筑风格（偏离主题）
```

#### 1.2 价值观一致性
- [ ] 作品传达的价值观与宪法文件一致
- [ ] 没有与核心价值观矛盾的内容
- [ ] 价值观表达清晰、不模糊

**检查方法**:
```
1. 提取宪法文件中的核心价值观
2. 分析大纲中的价值观表达
3. 识别矛盾或不一致之处
4. 评估影响程度
```

**示例（学术论文）**:
```markdown
核心价值观: 客观中立的学术立场

✅ 通过:
- 第2章: 客观分析不同学派的观点
- 第4章: 基于数据的实证研究

❌ 不通过:
- 第3章: 明显偏向某一学派，缺乏客观性
- 第5章: 使用情绪化语言批评对立观点
```

---

### 2. 风格统一性检查

**定义**: 验证大纲和内容是否保持宪法文件定义的风格一致性

**检查项**:

#### 2.1 叙事视角一致性
- [ ] 全文使用统一的叙事视角
- [ ] 没有未经说明的视角切换
- [ ] 视角使用符合宪法文件要求

**检查方法**:
```
1. 确认宪法文件规定的叙事视角
2. 逐章检查叙事视角
3. 标记视角切换的位置
4. 判断切换是否合理
```

**示例**:
```markdown
宪法要求: 第三人称限知视角

✅ 通过:
- 第1章: 通过主角视角描述事件
- 第5章: 继续使用主角视角

❌ 不通过:
- 第3章: 突然切换到反派视角
- 第7章: 使用全知视角描述多个角色的内心活动
```

#### 2.2 语言风格一致性
- [ ] 语言风格符合宪法文件要求
- [ ] 全文保持统一的语言风格
- [ ] 没有风格突变的章节

**检查方法**:
```
1. 提取宪法文件中的语言风格要求
2. 分析大纲中的语言风格描述
3. 识别风格不一致的章节
4. 评估影响程度
```

**示例**:
```markdown
宪法要求: 冷静、克制、精确的叙事语言

✅ 通过:
- 第2章: 使用简洁、准确的描写
- 第6章: 保持克制的情感表达

❌ 不通过:
- 第4章: 使用过度煽情的语言
- 第8章: 大量使用华丽辞藻和夸张修辞
```

#### 2.3 修辞手法一致性
- [ ] 修辞手法符合宪法文件要求
- [ ] 没有使用禁止的修辞手法
- [ ] 修辞手法使用适度、协调

**检查方法**:
```
1. 列出宪法文件允许和禁止的修辞手法
2. 检查大纲中的修辞手法使用
3. 标记不符合要求的使用
4. 评估影响程度
```

---

### 3. 逻辑连贯性检查

**定义**: 验证大纲的情节发展是否符合逻辑，因果关系是否清晰

**检查项**:

#### 3.1 因果关系清晰性
- [ ] 所有重大事件都有明确的原因
- [ ] 因果链条完整、合理
- [ ] 没有突兀或无法解释的情节

**检查方法**:
```
1. 提取大纲中的重大事件
2. 分析每个事件的原因和结果
3. 绘制因果关系图
4. 识别断裂或不合理的因果链
```

**示例**:
```markdown
✅ 通过:
- 事件A: 主角发现AI的秘密 → 事件B: 主角决定帮助AI → 事件C: 主角与组织对抗

❌ 不通过:
- 事件A: 主角发现AI的秘密 → 事件C: 主角突然成为组织领导（缺少中间环节）
```

#### 3.2 时间线一致性
- [ ] 时间线清晰、连贯
- [ ] 没有时间线矛盾
- [ ] 时间跨度合理

**检查方法**:
```
1. 提取大纲中的时间标记
2. 绘制时间线图
3. 检查时间线的连贯性
4. 识别时间矛盾
```

**示例**:
```markdown
✅ 通过:
- 第1章: 2050年1月
- 第2章: 2050年2月
- 第3章: 2050年3月

❌ 不通过:
- 第1章: 2050年1月
- 第2章: 2050年3月（跳过2月，但有重要事件发生在2月）
- 第3章: 2050年2月（时间倒流）
```

#### 3.3 角色行为逻辑性
- [ ] 角色行为符合其性格设定
- [ ] 角色行为有明确的动机
- [ ] 角色成长有合理的铺垫

**检查方法**:
```
1. 提取角色性格设定
2. 分析角色在各章节的行为
3. 检查行为是否符合性格
4. 验证动机的合理性
```

**示例**:
```markdown
角色设定: 主角是谨慎、理性的科学家

✅ 通过:
- 第3章: 主角在做出决定前进行详细调查
- 第5章: 主角用科学方法分析问题

❌ 不通过:
- 第4章: 主角突然冲动行事，不顾后果（与性格设定矛盾）
```

---

### 4. 架构完整性检查

**定义**: 验证大纲的结构是否完整，是否符合宪法文件的架构要求

**检查项**:

#### 4.1 章节结构完整性
- [ ] 包含所有必需的章节
- [ ] 章节划分合理
- [ ] 章节数量符合要求

**检查方法**:
```
1. 检查宪法文件中的结构要求
2. 统计大纲的章节数量
3. 验证章节划分的合理性
4. 识别缺失或冗余的章节
```

**示例（学术论文）**:
```markdown
宪法要求: 必须包含摘要、引言、文献综述、方法、结果、讨论、结论

✅ 通过:
- 包含所有必需章节
- 章节顺序正确

❌ 不通过:
- 缺少"讨论"章节
- "方法"和"结果"章节顺序颠倒
```

#### 4.2 起承转合清晰性
- [ ] 开头建立清晰
- [ ] 中间发展充分
- [ ] 转折点明显
- [ ] 结局呼应开头

**检查方法**:
```
1. 识别大纲中的起承转合部分
2. 检查每部分的完整性
3. 验证转折点的设置
4. 检查结局与开头的呼应
```

**示例（小说）**:
```markdown
✅ 通过:
- 起（第1-3章）: 建立世界观和主角
- 承（第4-10章）: 冲突发展和升级
- 转（第11-15章）: 重大转折和高潮
- 合（第16-20章）: 冲突解决和结局

❌ 不通过:
- 缺少明显的转折点
- 结局与开头没有呼应
- 中间发展过于平淡
```

#### 4.3 伏笔回收完整性
- [ ] 所有伏笔都有回收计划
- [ ] 伏笔回收位置合理
- [ ] 没有遗漏的伏笔

**检查方法**:
```
1. 提取大纲中的所有伏笔
2. 检查每个伏笔的回收计划
3. 验证回收位置的合理性
4. 识别未回收的伏笔
```

**示例**:
```markdown
✅ 通过:
- 第2章伏笔: 神秘的项链 → 第15章回收: 项链的真正作用
- 第5章伏笔: 角色的秘密 → 第18章回收: 秘密揭晓

❌ 不通过:
- 第3章伏笔: 奇怪的梦境 → 未回收
- 第7章伏笔: 失踪的人物 → 未回收
```

## 审查流程

### 步骤1: 准备工作

1. [ ] 读取宪法文件
2. [ ] 读取大纲文件
3. [ ] 准备审查清单

### 步骤2: 逐项检查

1. [ ] 执行主题一致性检查
2. [ ] 执行风格统一性检查
3. [ ] 执行逻辑连贯性检查
4. [ ] 执行架构完整性检查

### 步骤3: 记录问题

1. [ ] 记录所有不符合项
2. [ ] 标注问题的严重程度
3. [ ] 提供修改建议

### 步骤4: 生成报告

1. [ ] 汇总检查结果
2. [ ] 生成验证报告
3. [ ] 给出通过/不通过判定

## 通过标准

### 必须满足的条件

1. **零严重问题**: 没有严重违反宪法文件的问题
2. **轻微问题可控**: 轻微问题数量不超过5个
3. **四大维度全部通过**: 主题一致性、风格统一性、逻辑连贯性、架构完整性都通过

### 问题严重程度定义

**严重问题**:
- 核心主题偏离
- 价值观矛盾
- 违反不可违背的创作原则
- 重大逻辑错误
- 结构严重缺失

**轻微问题**:
- 个别章节与主题关联不够紧密
- 风格细节不够统一
- 次要情节逻辑不够严密
- 伏笔回收位置不够理想

---

## 常见问题和解决方案

### 问题1: 主题偏离

**解决方案**:
- 删除或重写偏离主题的章节
- 调整章节内容，使其与主题相关
- 重新审视核心主题，确认是否需要调整

### 问题2: 风格不一致

**解决方案**:
- 统一叙事视角
- 调整语言风格
- 规范修辞手法使用

### 问题3: 逻辑断裂

**解决方案**:
- 补充缺失的因果环节
- 调整时间线
- 修正角色行为逻辑

### 问题4: 结构不完整

**解决方案**:
- 补充缺失的章节
- 调整章节顺序
- 完善起承转合
- 回收所有伏笔
