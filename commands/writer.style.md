# writer.style - 风格分析与定义

## 命令描述

分析现有文本的风格或为项目定义新的风格指南。

---

## 用户输入

请提供以下信息:

1. **项目路径**: [project-path]
2. **操作类型**: 
   - analyze (分析现有文本风格)
   - define (定义新风格)
3. **参考文本**: [如果是analyze,提供文本路径或内容]
4. **风格要求**: [如果是define,描述期望的风格]

---

## 执行大纲

### 步骤1: 执行风格操作

**如果是analyze**:
- 调用 `style-analyzer` Agent
- 分析文本的语调、用词、句式等特征
- 生成风格报告

**如果是define**:
- 根据用户要求创建风格指南
- 包含语调、人称、时态、术语等规范

### 步骤2: 保存风格文档

将风格文档保存到 `writer_style/[project-name]_style.md`

### 步骤3: 输出结果

```
[完成] 风格文档已创建
文件路径: writer_style/[project-name]_style.md

风格特征:
- 语调: [tone]
- 人称: [person]
- 时态: [tense]

下一步建议:
在写作时参考此风格文档保持一致性
```
